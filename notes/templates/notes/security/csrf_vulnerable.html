What is CSRF?

CSRF is an attack where a malicious website tricks you into performing unwanted actions on another site where you're logged in. The attacker exploits the fact that your browser automatically sends cookies with requests.

Vulnerable Code Pattern:

# VULNERABLE: No {% csrf_token %} check
def csrf_vulnerable_add(request):
    if request.method == 'POST':
        # No CSRF validation
        Note.objects.create(title=title, description=description)

How CSRF Attack Works:

Step 1: You visit this website and stay logged in

Step 2: You click a malicious link that goes to a different website

Step 3: That website has hidden code that submits a form to THIS website

Step 4: Your browser automatically includes your session cookie

Step 5: The note gets created without your knowledge!

Malicious HTML (What hackers would create):

<!-- Hosted on attacker.com -->
<form action="http://yoursite.com/security/csrf/vulnerable/" method="POST">
    <input type="hidden" name="title" value="Hacked Note">
    <input type="hidden" name="description" value="Your account was compromised">
    <input type="submit" value="Click to claim prize!">
</form>

Why CSRF is Dangerous:

  • Unauthorized Actions: Attackers can modify or delete your data
  • Financial Fraud: Transfer money, make purchases
  • Account Takeover: Change password, email, settings
  • Reputation Damage: Post harmful content on your account
  • Silent Attacks: User never knows what happened

Requirements for CSRF to Work:

  1. User must be logged into the target site
  2. User must visit a malicious website
  3. The target site must not validate CSRF tokens
  4. The action must work with a simple POST request
