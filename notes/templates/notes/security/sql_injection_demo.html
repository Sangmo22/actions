🚨 SQL Injection - VULNERABLE Version

VULNERABILITY STATUS: 🔴 VULNERABLE
Issue: This page constructs SQL queries using string concatenation, allowing SQL code injection.

How This Works:

The vulnerable code directly concatenates user input into SQL queries without parameterization, allowing attackers to inject arbitrary SQL commands.

Vulnerable Code Pattern:

sql = f"""
INSERT INTO notes_note (title, description)
VALUES ('{title}', '{desc}');
"""
cursor.execute(sql)

Why This is Dangerous:

The quotes in the SQL string can be closed by the attacker's input, allowing them to inject additional SQL commands.

Try These SQL Injection Payloads:

Payload 1 (Data Extraction):
Title: Normal'; " or "1"="1
Description: This will extract all data

Payload 2 (Comment-based bypass):
Title: Test'); --
Description: Comment syntax injected (min 10 chars)

Payload 3 (Union-based):
Title: Test' UNION SELECT username, password FROM auth_user --
Description: This would extract user credentials if successful

Payload 4 (Time-based blind):
Title: Test'; WAITFOR DELAY '00:00:05'; --
Description: This would delay the response by 5 seconds

What Happens with SQL Injection:

If you submit Title: Test'); DROP TABLE notes_note; --

The SQL becomes:

INSERT INTO notes_note (title, description)
VALUES ('Test'); DROP TABLE notes_note; --', '...')

This executes TWO commands:

  1. INSERT with title 'Test'
  2. DROP TABLE notes_note (deletes all data!)
  3. The rest is commented out with --

Common SQL Injection Impacts:

  • Data Breach: Extracting sensitive information
  • Data Modification: Changing or deleting records
  • Authentication Bypass: Bypassing login checks
  • Privilege Escalation: Gaining admin access
  • Code Execution: Running OS commands (depending on DB permissions)
