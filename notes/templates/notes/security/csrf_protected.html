PROTECTION STATUS: 🟢 PROTECTED
Solution: This page uses Django's CSRF token validation to ensure requests originate from trusted sources.

How CSRF Protection Works:

A CSRF token is a unique, secret value that:

  • Is generated per user session
  • Is included in every form as a hidden field
  • Is validated on the server before processing POST requests
  • Cannot be obtained by cross-origin requests

Protected Code Pattern:

<form action="..." method="post">
    {% csrf_token %}
    <input type="text" name="title">
</form>

What Actually Gets Rendered:

<input type="hidden" name="csrfmiddlewaretoken" value="9a...kK">

🔐 CSRF Token Validation Flow:

  1. User visits form: Server generates unique CSRF token for this session
  2. Token in form: Hidden field with the token is included in the HTML
  3. User submits: Browser includes the token with the POST data
  4. Server validates: Compare received token with stored session token
  5. If valid: Process the request normally
  6. If invalid/missing: Reject the request with 403 Forbidden error

Why CSRF Attacks Fail Here:

Scenario: Attacker tries to forge a request

An attacker creates a hidden form on their malicious website:

<form action="https://yoursite.com/security/csrf/protected/" method="POST">
    <input type="hidden" name="title" value="Hacked">
    <input type="hidden" name="description" value="Exploit">
</form>

Problem for attacker:

  • The form is missing the CSRF token
  • The attacker cannot obtain your unique CSRF token (same-origin policy)
  • Even if they guess a token, it won't match YOUR session token
  • Server rejects the request with 403 Forbidden
  • Attack fails silently from the user's perspective

Django CSRF Middleware Verification:

When Django receives a POST request:

1. Extract CSRF token from request (POST data or header)
2. Verify token exists
3. Compare with token in user's session
4. If match: ✅ Process request
5. If no match: ❌ Return 403 Forbidden

Best Practices for CSRF Prevention:

  1. Always include {% csrf_token %}: In every form that modifies data
  2. Use POST/PUT/DELETE: Never use GET for state-changing operations
  3. Enable CSRF middleware: Keep 'django.middleware.csrf.CsrfViewMiddleware' active
  4. Use secure cookies: Set Secure and HttpOnly flags on cookies
  5. SameSite cookies: Add SameSite=Strict/Lax attribute
  6. Custom AJAX requests: Send CSRF token in headers

AJAX Example with CSRF Token:

// Get CSRF token from page
const token = document.querySelector('[name=csrfmiddlewaretoken]').value;

// Send with fetch
fetch('/api/notes/', {
    method: 'POST',
    headers: {
        'X-CSRFToken': token
    },
    body: JSON.stringify({title: 'Test', description: 'Safe'})
})
